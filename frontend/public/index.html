FILTER

Set up conditions on which to filter out entries in your results. For conditions on string values, wildcards can be specified using * before and/or after the value.

Inverting the condition takes the opposite of what is entered (i.e. NOT of condition)

Note that specifying more than one condition here will be considered conditions that must be met at the same time (i.e. Condition1 AND Condition2).
As such, it is recommended that a single column only be involved in no more than one condition.

The comparison operator "IS" can only be used with non-numerical fields / columns.

Conditions will only be considered for those that specify a column to check, have a filtering operator, and a non-empty value.

Select the columns to be displayed:

Do you need your results to be sorted?

Select the direction to sort the results:

Select the columns on which to sort. With each selected column, specify the order number for it. Ties in earlier keys will be broken by values in later ones (low to high).

Ensure that values in the order numbers are distinct. They do not have to be 1,2,3, etc. ; however you must make sure keys to sort on first are lower valued and subsequent ones are higher.

Aggregated Columns

Does the output need to be grouped?

Select the columns on which to create groupings:

Create any aggregated columns here. Specify a custom name for your column, the aggregation type to run, and the column on which to run it.

Aggregations will only be created where column names are defined, selecting an aggregation type without a name has no effect.
Be mindful of the type of dataset being queried, as it will affect what columns can be used in an aggregation.

Aggregated columns are only available when Transformations - Grouping is enabled.
